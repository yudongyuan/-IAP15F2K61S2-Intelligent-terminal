🏆 IAP15F2K61S2-Intelligent-terminal

基于 IAP15F2K61S2 的智能超声波测距与波形回放终端 (蓝桥杯单片机组实战总结)

💡 核心代码说明：详细的代码逻辑和模块注释讲解，请直接查看仓库内的 蓝桥杯基本模块构建/蓝桥杯基本模块构建.c 文件。

✨ 项目核心功能与技术栈

高精度测距与校准：编写 DS18B20 单总线驱动，设计温度补偿测距算法，测距精度稳定在 ±3cm 以内，校准值可动态调整。

可靠数据采集：构建循环缓冲区，每 50ms 采样一次距离值；通过按键触发，期间屏蔽其他按键操作以保证数据完整性。

模拟波形回放：编写 I²C 驱动控制 PCF8591 DAC 模块，按原始采样间隔将缓冲区数据还原为模拟电压输出，实现波形回放。

PC 端可视化验证：通过 UART 读取缓冲区距离数据，编写 Python 脚本使用 matplotlib 绘制距离-时间曲线，完成波形回放的闭环验证。

掉电存储机制：利用 SPI 接口扩展外部 Flash 存储芯片，实现多组记录数据的掉电保存与读取，提升数据可靠性。

底层并发与调优：通过优化定时器中断，采用状态机消抖，最终测距数据刷新率稳定在 ≤1s，按键响应时间 ≤0.1s。

📖 备赛思路与时间线建议

大三下学期课程通常比较紧张，如果你有拿奖的决心，强烈建议在寒假结束前就开始刷省赛真题，并且学会自己总结一套代码模板！以下是我推荐的通关路线：

📅 备赛时间轴

1月寒假前 — 2月开学：完成所有外设模块的初步学习。底线是每个模块要能独立跑通！

推荐资源：B站搜 “小蜜蜂老师” 的视频快速过一遍基础。

避坑指南：不用学他的编程方式！可以学西风的课程，需要一套高度模板化的写法。

进阶资源：B站搜 “西风” 的课程，重点学习他那种模板化的编程框架。

2月开学后 — 3月末：开始狂刷真题。至少完成全部省赛真题一遍，对客观题型有充分的了解，并在实践中完成个人模板的总结。

3月末 — 比赛前：开始刷国赛题。越多越好，最好全过一遍。

🛠️ 关于“总结代码模板”的经验
比赛总共就 5 个小时，其中还包括做客观题的时间。你唯一的目标就是：快速且正确地完成编程。

蓝桥杯单片机组的硬件板子是要熟悉的，平时训练一定要记录自己常犯的错。总结模板的核心目的只有两个：

杜绝低级错误：避免因为查错浪费大量时间导致做不完（省赛只要编程题做不完，大概率和省一无缘）。

培养查错直觉：比赛中遇到 Bug，能凭借经验快速定位是哪个模块出了问题。

⚠️ 建议：上考场前一定要有带有你个人编程风格的模板（官方会查重！），然后像默写一样高效敲代码。千万不要看题目要求遇到一个模块就现场去思考现写一个模块，那样绝对来不及。

🎯 刷题优先级与比赛趋势
以前的老题目考法跟现在不太一样了，精力有限的话，请严格按照以下优先级刷题：
省七以后第一遍 > 国七以后第一遍 > 省七以后第二遍 > 各类模拟题

📌 关于比赛趋势：
去年的情况是“省赛极难，国赛一般”。所以，在打省赛之前先去做几套往年的国赛真题，会对你过省赛有极大的帮助。

📊 真实的获奖“及格线”参考
以下国赛评估基准，整套卷子大概会考察 15 - 25 个功能点。

客观题在省赛里用来拉分很关键，但在国赛里其实没那么要命（我一朋友国赛客观题只对了一个，最后拿了国一 🤣）。
以我个人拿省一时的成绩为例：编程题实现 95%（全部功能写完，但有一处 bug），客观题 7/10，在省一中排名中上。

结合经验，以下是我认为的最低获奖参考线：

🥇 省一最低标准：编程题差 1 个功能未做出 + 客观题 4/10。

🥈 国二最低标准：编程题差 2 个功能未做出 + 客观题 3/10。

🏆 国一最低标准：编程题 100% 全部完成 + 客观题 1/10。
